# Harenohi UX Guidelines

> リサーチエージェント + 実ユーザーテスト（2026-04-14）のフィードバックを統合。実装時に必ず参照すること。

---

## 1. Mobile Interaction Fundamentals

### Touch Targets
- 全インタラクティブ要素: **最小44×44px**（Apple HIG）、推奨48px
- 星評価: 各星48px、星間隔8px以上（横幅合計280px以上）
- チップ/タグ: `min-h-[44px] px-4`
- リンクテキスト: `py-2 block` で高さ確保

### Tap Feedback（全タップに即時反応）
- カード: `active:scale-[0.98]` + `transition-all duration-[400ms]`
- ボタン: `active:scale-95` + `transition-all duration-[400ms]`
- ナビリンク: `active:bg-muted`
- 星評価: `active:scale-90`
- 300ms遅延排除: `touch-action: manipulation` on html

### Animation Timing (v3 — Luxury Standard)
- **全般**: 早すぎるアニメーションはチープに見える。ラグジュアリー = ゆっくり
- ページ/セクション入場: duration **0.9-1.0s**, ease `[0.16, 1, 0.3, 1]`
- スタガー遅延: **0.15-0.2s** per item
- Spring: stiffness **100-140**, damping **18-22**
- CSS transition: **400ms** (`duration-[400ms]`)
- ホバー/タップ反応: **400ms** ease-out
- 参考: Aesop ~600ms、Apple ~500ms。150msはNG
- `-webkit-tap-highlight-color: transparent`

### SafeArea
- ボトムナビ: `pb-[env(safe-area-inset-bottom)]`
- レイアウト: `pb-[calc(4rem+env(safe-area-inset-bottom))]`
- ヘッダー: `pt-[env(safe-area-inset-top)]`（将来のPWA対応時）

## 2. Loading & Feedback Patterns

### Loading States（パターン選択基準）
| パターン | 使用場面 |
|---------|---------|
| **スケルトン** | コンテンツ一覧、カード、テーブル（構造が予測できるもの） |
| **ボタンスピナー** | ボタンのServer Action実行中（2〜10秒） |
| **プログレスバー** | ファイルアップロード、10秒超のAI処理 |

- 全ページに `loading.tsx` を配置。スケルトンはページのレイアウト構造に合わせる
- フィルタ・検索・ソートはクライアントサイド処理（サーバー往復なし）

### Toast Notifications（Sonner）
- ライブラリ: **Sonner**（shadcn/ui公式統合）
- 位置: モバイルは**下部**、デスクトップは**右下**
- 表示時間: 確認メッセージ2〜3秒、アクションボタン付き5秒
- 同種通知は置換（スタックしない）
- `role="status"` + `aria-live="polite"`

### Success Feedback
- 保存成功: ボタンに✓チェックマーク表示（800ms）→ 元に戻る
- お気に入り: ハートアニメーション（scale 1→1.3→1.0, 200ms）
- 最終決定: confetti（canvas-confetti, particleCount: 100）— **この1箇所のみ**
- `prefers-reduced-motion` を必ず尊重

### Error Handling
- フィールドバリデーション → インライン（onBlur タイミング）
- Server Action失敗 → トースト（リトライボタン付き）
- ページレベルエラー → `error.tsx` Error Boundary
- オフライン → 持続トースト（自動dismissしない）
- エラーメッセージ: 「何が失敗したか」+「次のアクション」を日本語で具体的に

## 3. Card & List Design

### 式場カード（Booking.com + ゼクシィ融合パターン）
- **写真ファースト**: 16:9 or 4:3のメイン写真を上部に配置（将来Phase対応）
- **情報密度は高めに**: 日本ユーザーは「情報量 = 信頼」。カードに価格帯・収容人数・エリア・スタイルタグを表示
- **スコア**: 右上に大きく表示（Booking.com方式）
- **ショートリスト**: カード右上にハートアイコン。タップで即時追加 + アニメーション

### リスト管理（大量アイテム対応）
- 検索: インスタントフィルタ（クライアントサイド）
- フィルタ: ステータスチップ（横スクロール）+ ボトムシート（詳細フィルタ）
- ソート: 登録順 / 名前順 / スコア順
- ページネーション: 10件以上はバーチャルスクロールまたは「もっと見る」

## 4. Comparison UX

### モバイル比較の原則（NN/g準拠）
- 375pxで並列比較は**最大2アイテム**。3つ目は横スクロール
- ヘッダー行（式場名）: sticky top
- 左端の属性列: sticky left（`min-w-[80px]`）
- 両軸スティッキーのハイブリッドが最適

### レーダーチャート
- モバイル: 高さ200px、outerRadius 60、ラベル9px
- デスクトップ: 高さ300px、outerRadius 90、ラベル11px
- 6軸: 雰囲気、ホスピタリティ、料理、費用、アクセス、総合印象

## 5. Rating Input

### 星評価
- 各星48px、`gap-2`（8px間隔）
- タップで即時自動保存（デバウンス300ms）+ 「取り消し」トースト3秒
- 各次元に1行ヘルプテキスト（例: 「チャペルや会場全体の雰囲気」）

### 評価軸（Tier 1 — 6次元）
| キー | 日本語ラベル | ヘルプテキスト |
|------|------------|--------------|
| atmosphere | 雰囲気 | チャペルや会場全体の雰囲気 |
| hospitality | ホスピタリティ | プランナーやスタッフの対応 |
| cuisine | 料理 | 試食の味・量・プレゼンテーション |
| cost | 費用 | 見積もりの納得感・コストパフォーマンス |
| access | アクセス | 駅からの距離・交通の便 |
| overall_impression | 総合印象 | 全体を通しての満足度 |

## 6. Japanese Wedding UX Cultural Patterns

### 採用すべきパターン
- **情報密度は高めに保つ**: 過度なホワイトスペースは「情報不足 = 信頼できない」
- **費用の早期表示**: 「お問い合わせください」は離脱要因。概算でも数字を出す（トキハナ方式）
- **急かさないUX**: フェア当日即決を前提としない。「じっくり比較しましょう」のトーン
- **UIコピーは丁寧体**: 「予約する」→「見学してみる」、「決定」→「この式場に決めましょう」
- **ユーザー投稿写真**: リアリティ担保。過度に加工された公式写真への不信感が強い

### 避けるべきパターン
- 過度なシンプルさ（欧米式の広いホワイトスペース）
- 費用情報の隠蔽
- 緊急性メッセージ（「残りわずか！」— Booking.com方式はウェディングに不適）
- ピンク一辺倒（ゼクシィとの差別化のためNavy + Goldを維持）

## 7. Empty States

### 構成要素（必須）
1. アイコン（Lucideから。イラストは不要）
2. 見出し（1文）
3. サポートテキスト（具体的な次のステップ）
4. **CTAボタン**（必ず1つ配置。テキストだけで終わらない）

### 例
```
[Search icon]
まだ式場が登録されていません
気になる式場を追加して、比較を始めましょう
[+ 式場を追加する] ← Button
```

## 8. Decision Ceremony

### 最終決定時の演出
- `canvas-confetti` (2KB) でコンフェッティ
- 全画面オーバーレイ（500ms fade-in）→ 式場名 + 決定日 + 理由
- 「最後まで迷った式場」と「決め手」を構造化記録
- スキップ可能（アニメーション5秒以内）
- `prefers-reduced-motion` 時はアニメーション省略、テキストのみ

## 9. Shortlist Discovery

### 導線（発見性の改善）
- 式場カードにハートアイコン（右上）→ タップで即ショートリスト追加
- ボトムナビの「候補」タブにバッジ（件数表示）
- ショートリスト空ステートに具体的手順:「式場一覧でハートをタップすると候補に追加されます」
- 式場詳細ページにも「候補に追加」ボタン（大きく、目立つ位置に）
