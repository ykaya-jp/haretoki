# Haretoki クラリティ向上計画 (Clarity Plan)

> 実ユーザーの声「このアプリ、何ができるのかよくわからない / どう使えばいいのかわからない」への応答。
> 作成日: 2026-04-14 / Status: Draft（実装前レビュー要）

---

## 1. 問題の再定義

ユーザーの「何ができるか分からない」を分解すると、以下の複合問題として現れている:

- **A. 価値訴求の抽象性**: ランディング冒頭「その直感、信じていい日にする。」は情緒的だが、アプリで何が起きるかは伝わらない。ヒーロー直下に機能Demoが無い。
- **B. Aha体験までの距離**: サインアップ → 4問の対話オンボ → 「おすすめ」表示、まで到達しないと価値を体感できない。途中離脱すると真っ白なホームに着地する。
- **C. Next Best Action の不在**: 認証直後のホームで `JourneyCard` の "empty" 状態は「式場を見てみる」一択。その先（評価→比較→決定）の流れが見えない。
- **D. 機能間リンケージ不足**: 「探す/候補/コーチ」各タブが独立して見え、"このアプリで一連の意思決定ができる" という全体像が立ち上がらない。
- **E. AIコーチの使い所が不明**: `CoachQuickStart` で3つの質問チップはあるが、"いつ、なぜ" 話しかけるかの文脈が提示されない。

## 3. 現行体験の問題ポイント（ファイル参照）

### 3-1. ランディング — `src/components/landing/landing-page.tsx`
- L112-132: ヒーローコピーが情緒的のみ。「見積もりが上がる」「AIが比較」「見学チェックリスト」などの **具体的ベネフィット** がファーストビュー外（`py-24` の Problem セクション以降）にある。
- L250-286: `How It Works` の3ステップは美しいが、**画面キャプチャ/UIモックが無い**ため「本当にURL貼るだけ?」の疑念が解消されない。
- CTA「無料ではじめる」は良いが、**"登録しないで体験" の入口がゼロ**。

### 3-2. オンボーディング — `src/components/onboarding/onboarding-flow.tsx`
- L30-78: 4問は丁寧だが、**冒頭に "このあと何が起きるか" の提示が無い**（例: 「3問お伺いします。最後におふたりに合う式場をAIがご提案します」）。
- L258-354: レコメンド結果画面に到達しても、「追加したあと何をすればいいか」のガイドが無い（評価/見学記録/比較の導線なし）。
- スキップ率が高い場合、空のホームに着地する（3-3へ）。

### 3-3. ホーム（認証後）— `src/app/(app)/home/page.tsx` + `journey-card.tsx`
- `journey-card.tsx:68-76` empty 状態: メッセージは「式場探しをはじめてみませんか」のみ。**このアプリで何ができるのか（比較・見学記録・AI相談・最終決定）が一切提示されない**。
- 初回ユーザーは `RecentVenues` の空ステート + Journey empty + インサイトなし の **三重の空っぽ** に遭遇。
- 4タブの役割説明がどこにも無い。

### 3-4. 各タブの文脈
- **Explore** (`explore/page.tsx:146-152`): 空ステート「右上の『追加』からURLを貼るだけ」は良い。ただし追加後に何が起きるかの予告が無い。
- **Candidates** (`candidates/page.tsx`): `<h2>候補</h2>` のみ。初回は空 → 何を候補にするのかの定義が示されない。
- **Coach** (`coach/page.tsx:22-36`): `CoachQuickStart` が3プロンプト用意するが、"なぜ今コーチと話すのか" の文脈が無い。初回の心理ハードルが高い。

## 4. 対策案（UX Interventions）

### 対策1. ランディング ファーストビューに "アプリで何ができるか" を 3 icon + 1行で明示
- **WHAT**: ヒーロー詩文の下に、横並び/縦3アイコン「① URL貼るだけで式場登録 ② 6軸×AIで比較 ③ 見学リストで当日迷わない」を配置。各8〜14字。
- **WHY**: 情緒コピー単体では機能理解が形成されない（Duolingo/Notion LP はヒーロー直下に機能スクショを置く定番）。
- **HOW**: `landing-page.tsx` L112-161 のヒーロー内、CTA直前に `<ul>` として挿入。既存 `FEATURES` から3件ピックして短縮。
- **Refero根拠**: Linear / Notion / Zola のランディングは "What you get in one sentence × 3" をヒーロー直下に置く共通パターン。
- **Effort**: **S**

### 対策2. ランディングに "60秒デモ動画 or 静的モックGIF" を追加
- **WHAT**: 「URL貼付 → カード生成 → 評価 → 比較 → 決定」を 40〜60秒の無音ループ動画 or 3枚の静的モック（swap on scroll）で見せる。
- **WHY**: `How It Works` の文字情報だけでは「本当に3分で始められる」が信じられない（対策1のベネフィットの **証明** に相当）。
- **HOW**: `landing-page.tsx` L250-286 の `How It Works` セクションを拡張。Next.js `<Image>` の連続表示 or MP4 (loop/muted/playsInline)。動画は `/public/demo/` に格納。
- **Refero根拠**: Cal.com / Superhuman / Arc LP は "実画面のループ動画" で機能理解を加速。
- **Effort**: **M**（動画制作コストを含む）

### 対策3. 認証直後に「3ステップ予告 + 最初のアクション提示」を行う薄いオーバーレイ
- **WHAT**: サインアップ直後、オンボ質問開始前に 1枚の歓迎シート:
  - 見出し「式場選びを、3ステップで。」
  - ① 好みを3問お伺いします
  - ② AIがおふたりに合う式場をご提案
  - ③ 気に入ったら評価・比較・決定まで、このアプリで
  - CTA「はじめる」「スキップ」
- **WHY**: 「何が起きるか分からないまま質問される」を解消（現行 onboarding-flow.tsx は突然 "どんな雰囲気がお好みですか?" で始まる）。
- **HOW**: `onboarding-flow.tsx` に `step === -1` の事前ステートを追加。または新規 `OnboardingIntro` コンポーネント。既存 Morning Light パレット + serif heading を流用。
- **Refero根拠**: Zola / Joy の couple app は "What to expect" スライドをオンボ冒頭に必ず入れる。
- **Effort**: **S**

### 対策4. ホームの `JourneyCard` empty 状態を "4ステップ可視化 + NBA常設" に再設計
- **WHAT**: 現行 empty 状態を以下に差し替え:
  - 見出し「おふたりの式場さがし」
  - 進捗ドット 4つ（追加 → 見学 → 比較 → 決定）すべて未達表示
  - 直下に **Next Best Action カード**「まず1件、気になる式場を追加してみましょう」+ CTA「式場を追加」「URL貼付で登録」2択
  - サブ文言「追加すると、AIが見積もりの注意点を先回りでお伝えします」
- **WHY**: 「何をすればいいか」「その先に何があるか」を同時に示す。曇り→晴れのメタファーを "4つの天気ステップ" で視覚化。
- **HOW**: `journey-card.tsx` L68-76 を拡張。新規 `<JourneySteps />` サブコンポーネントを追加（4つの ☁ → ⛅ → ⛅ → ☀ ドット）。ブランドメタファー "曇り→晴れ間→晴れ" と完全整合。
- **Refero根拠**: Linear onboarding / Notion template gallery は "dotted progress + next action" を初期ホームの hero に置く。
- **Effort**: **M**

### 対策5. AIコーチにタブ冒頭 "用途カード" を設置（いつ・なぜ話しかけるかを明示）
- **WHAT**: `/coach` を開いたときの最上部に、状況別の会話入口カード3枚:
  - 「式場選びの基準がまだ決まっていない」→ 好みヒアリング
  - 「見積もりの妥当性が不安」→ 見積もり診断（式場未登録でも相場ガイド）
  - 「ふたりで意見が割れている」→ 意思決定サポート
- **WHY**: 現行の `CoachQuickStart` (coach-quick-start.tsx:8-12) は質問リストのみで、**ユースケース文脈が欠落**。カップルは "自分の悩みに該当するか" で判断する。
- **HOW**: `coach-quick-start.tsx` を拡張し、3カード形式（`gold-subtle` 背景 + serif title + 1行 desc + chips）に。既存 AI Insight Card のスタイルトークンを流用。
- **Refero根拠**: Intercom / Superhuman の AIアシスタント初期画面は "ユースケース別カード" 構成が標準。
- **Effort**: **S-M**

### 対策6. 全タブのヘッダー下に "このタブでできること" 1行サブタイトルを追加
- **WHAT**: 例:
  - 探す: 「URL貼付 or 検索で、気になる式場を集める場所」
  - 候補: 「お気に入りを並べて、ふたりで比べる」
  - コーチ: 「迷ったら、AIに聞ける相談相手」
  - ホーム: 「今日のおふたりの式場選び」
- **WHY**: タブ名だけでは意図が伝わらず、各タブが独立機能に見える（問題D）。
- **HOW**: 各 `page.tsx` の `<h2>` 下に `<p className="text-sm text-muted-foreground">` を追加。コピーは DESIGN.md 準拠の丁寧体・静謐トーン。
- **Effort**: **S**

## 7. 非機能・ブランド整合

- **モバイル375px**: 対策1の3ベネフィットは縦積み（iconサイズ h-6）、対策4の進捗4ドットは `gap-2` で375px内に収まる設計。
- **ラグジュアリー**: 細字 Noto Serif JP 見出し、太字禁止、`tracking-[0.06em]` 遵守。対策2の動画はミュート + ループ、派手なアニメなし。
- **DESIGN.md 整合**: Gold subtle / warm、Morning Light パレット、`active:scale-[0.98]` フィードバック、`h-11` タッチターゲットを全対策で遵守。
- **パフォーマンス**: 対策2の動画は < 500KB (webm) / `loading="lazy"` / `prefers-reduced-motion` で静止画にフォールバック。
- **a11y**: NBA ボタンに aria-label、進捗ドットに `role="progressbar" aria-valuenow`。
