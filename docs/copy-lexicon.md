# Haretoki コピー辞書 — "素敵化" 置換表

**目的**: 機械的/業務的に見える単語を、**ふたりが読んで温かく感じる表現**に置き換える。
**運用**: Sprint 5（B-22 全コピー素敵化）でこの表に沿って grep 置換 + 手動確認する。マーケティング用語（ランディング・Overview）とは区別。

最終更新: 2026-04-15

---

## 原則（Tone of Voice）

- **丁寧だが堅苦しくない**: 「予約する」→「見学してみる」、「保存」→「残す」
- **ふたり主語**: 「ユーザー」「あなた」ではなく「ふたり」「おふたり」
- **売らない**: 「おすすめ！」「今だけ！」は使わない
- **急かさない**: 「すぐに」「今すぐ」は避け、「ゆっくり」「あとから」を含める余地を残す
- **晴れの日メタファー**: 曇り（不安）→ 晴れ間（見えてきた）→ 晴れの日（確信と喜び）。文章内に光・空・朝の語彙をさりげなく
- **エンジニア用語の露出ゼロ**: 「プロジェクト」「タスク」「ログ」「セッション」「データベース」 → UI では隠す

## 置換表（UI 向け）

### 1. エンジニア/業務ワード

| Before | After | 備考 |
|---|---|---|
| プロジェクト | おふたりの式場さがし / 式場さがし | 意味上「Project」だが UI では隠す |
| プロジェクトに参加 | おふたりの式場さがしに合流する | 招待 email / accept-invite 画面 |
| プロジェクトオーナー | おふたりの最初のメンバー / はじめに登録した方 | エラーメッセージ |
| タスク / TODO | 次の一歩 / やること | 進捗系 |
| セッション（チャット） | これまでの会話 / 会話履歴 | 既に一部適用済（Sprint 2） |
| セッションを作成 | 新しい会話をはじめる | |
| ログ | 記録 / メモ | |
| データ | 情報 / 記録 / 中身 | |
| 削除 | 手放す / 消す | 決定取消・式場削除等 |
| 編集 | 書き直す / 手直しする | |
| 保存する | 残す | 「この想いを残す」等で既に一部適用 |
| 作成 | はじめる / つくる | 「新しく作成」→「あたらしく はじめる」 |
| 登録 | 置く / 迎える | 式場追加 Sheet で適用済 |
| 設定 | 整える / 決める | 「チェックリスト設定」→「何を比べるか、決める」（既に適用済） |
| 更新 | 手を入れる / 書き換える | |
| 読み込み中 | いま準備しています | skeleton 時の aria-label |
| エラー | うまくいきませんでした / また今度 | 既存 toast |
| バリデーションエラー | （文脈に応じて具体に）例: 「お名前を入れてください」 | |

### 2. 機械的なボタン文言 → 動作を具体に

| Before | After |
|---|---|
| 送信 | 送る / 伝える |
| 決定 | これに決める / この式場にする |
| キャンセル | やめる / 戻る |
| OK | わかった / はい |
| 確認 | たしかめる |
| 次へ | 次を見る |
| 戻る | ひとつ戻る |
| 閉じる | 閉じる（OK、これは変えない） |
| 取消 | 戻す / 取り消す |

### 3. 状態ラベル

| Before | After |
|---|---|
| 候補（データモデル上は維持）| 気になる / 見ている | UI は「気になる」で統一 |
| お気に入り | 本命 | ハート UI。実装の `VenueFavorite` は維持 |
| 評価済み | 印象メモ / 印象を残した | |
| 未評価 | まだ印象を残していない | |
| 決定済み | 晴れの日、決まり | |
| 見学予定 | 見学を入れた / 見学の日が決まっている | |
| 見学完了 | 見に行った / 印象を残した | |

### 4. 時制・トーン

| Before | After |
|---|---|
| いつでも | ゆっくりで大丈夫 |
| すぐに | （削る or「気が向いたときに」） |
| 必須 | あとで大丈夫。でもあると便利 |
| 必ず入力してください | 埋めてもらえると助かります |
| ログインしてください | おふたりのページに入る |
| アカウント作成 | はじめる / ふたりの場所をつくる |

### 5. 絵文字・記号の運用

- **絵文字は使わない**（👑 も撤去済み、Sparkles は lucide icon で表現）
- 「★」は評価表示のみ可。トースト・見出しでは使わない
- 「•」中黒、「→」右矢印、「·」中点、「◎」「✦」はアクセントとしてのみ OK

## 文体テンプレ（書き下ろし時の目安）

### 成功トースト
- ✗「保存しました」
- ✓「残しました」「メモを書き留めました」「記録に残しました」

### 失敗トースト
- ✗「エラーが発生しました」「失敗しました」
- ✓「うまくいきませんでした」「また今度試してみてください」

### 空ステート
- ✗「データがありません」「〇〇はまだありません」
- ✓「まだ 〇〇 を置いていません。最初の 1 件、追加してみませんか？」
  （Drop Zone + Inline Action で招待状にする／CLAUDE.md P1 原則）

### 確認ダイアログ
- ✗「本当に削除しますか？」
- ✓「この式場、候補から外しますか？ あとからまた追加できます。」
