## R-1 進捗リング → 天気アイコン化

**優先度**: H（E-1 と統合必須） / **工数**: 1 時間
**対象**: `editorial-hero.tsx` の JourneyRingSm

### What
percentage 表示を廃止 → **天気アイコン** に。
- 0-25%: ☁️ 曇り
- 25-60%: 🌤 晴れ間
- 60-90%: ☀️ 晴れ
- 100%: 🌞 よく晴れた日

「決定まで N%」の代わりに「ふたりの空、今日は◯◯」

### Why
- パーセントは抽象的で無味乾燥
- ブランドメタファーと完全一致
- リングの数値が「残り」を感じさせる → 天気なら「いま」を感じる

### How
EditorialHero の JourneyRingSm を削除、代わりに SkyChip の大型版 (48px) を中央に配置し、下に「今日は 晴れ間」のキャプション。内部ロジック同じ、UI だけ差し替え。

## R-2 探すフィルタ → "気分で探す" モード

**優先度**: M / **工数**: 3 時間
**対象**: `/explore` の VenueFilterSheet

### What
既存のフィルタ（エリア / 予算 / スタイル）に加えて、**情緒語タグ** の新セクション:
- 「光がきれい」「緑が見える」「静かな空間」「賑やかな雰囲気」「海が近い」「山並み」「都心の夜景」「歴史を感じる」

タップすると Venue.features 和訳で該当式場だけを絞る。

### Why
- 条件を "5W1H" で詰める理性的検索と違う、**気分ベース**の入口
- Airbnb "Categories" 型の新しい体験導入
- ユーザーが言語化できていない好みを引き出す

### How
- Venue に `vibeTags String[]` 追加（fit-reason 生成時に AI が同時に推定して埋める）
- VenueFilterSheet にタブ or セクション追加: `条件で絞る` / `気分で探す`
- 気分タグは最大 3 選

## R-3 式場詳細写真 → 朝/昼/夕切替

**優先度**: M / **工数**: 2 時間
**対象**: `PhotoCarousel` on venue detail

### What
写真ギャラリーに `朝 · 昼 · 夕` のセグメントタブ。タップで該当時間帯の写真だけ表示。光の変化を視覚的に理解できる。

### Why
- 挙式時間を決める時の判断材料
- 「光の変化」というブランドメタファーに深く整合
- ウェディングの視覚体験そのものが「時間の流れ」

### How
- Venue.photoUrls を `{ url, tod: "morning"|"noon"|"evening"|null }[]` に拡張（タグ付け機能）
- ユーザーが追加する写真に時刻メタデータを自動付与（EXIF から）
- 初回は None のまま、AI が「明るさ + 色温度」で自動分類

## R-4 AI insight → Expire 設計

**優先度**: H / **工数**: 2 時間
**対象**: `AIInsightCard` on home

### What
insight カードが生成から 3 日経つと **opacity 70% に褪色 + "もう一歩進みました" ラベル**。7 日で archive タブに移動（見えなくなる）。新しい insight は毎日のリチュアル (E-1) と連動生成。

### Why
- 現状、同じ insight が毎日ずっと並ぶ → "AI が働いていない" 印象
- "鮮度" を可視化することで毎日開く動機
- archive は見返せる → 過去の AI からの気づきが資産化

### How
- `AiAnalysis.createdAt` を利用して `getAIInsights()` で age を計算
- UI で age >= 3 days は `opacity-70 grayscale-[30%]`
- age >= 7 days は `/coach/history` に移動

## R-5 コーチ空状態 → "今夜の一問" 常設

**優先度**: M / **工数**: 2 時間
**対象**: `CoachQuickStart` 空状態

### What
QuickStart の上に、**毎日 AI が生成する "今夜の 1 問"** を添える:
> 「今夜、もし一言だけふたりで話すなら — 料理と雰囲気、どちらを優先したいですか？」

タップで /coach にそのまま送信、対話開始。

### Why
- 「何を話したらいいか分からない」→ 毎晩 1 問で関係性が深まる
- E-1 DailyRitual の夜バージョン
- コーチ機能の使用率 up

### How
- Cron で毎日 21:00 JST 生成（E-1 と同じ Cron で）
- `DailyCoachQuestion` model（簡易）
- CoachQuickStart の最上部に大きな 1 問カード

## R-6 比較ボード → 感情カラム追加

**優先度**: M / **工数**: 2 時間
**対象**: DecisionMatrix

### What
既存の 6 観点スコア (料理 / 会場 / etc) の下に、**"見学後の気持ち"** カラム追加:
- ☀️ 晴れやか / 🌤 明るめ / ☁️ もやもや
- VisitRating の `way_home_mood` (E-3 連携) を表示

### Why
- 数字スコアだけだと「理性的比較」で終わり、決められない
- 感情記録が数字と並ぶと、**認知的不協和** が可視化される
- 「スコア 4.5 だけど気持ちは ☁️ — なぜだろう」が決定の鍵

### How
- DecisionMatrix に新しい row 追加: "見学後の気持ち"
- E-3 way_home_mood と連動（未記録なら「— 帰り道モードで残しましょう」）

## R-7 マイページ → "二人の合意事項" セクション

**優先度**: M / **工数**: 4 時間
**対象**: `/mypage`

### What
マイページに新セクション「ふたりが決めたこと」:
- 決まった (Decision)
- 保留 (WIP)
- 要相談 (Partner 間で温度差)

AI が自動分類 or 手動で追加可能。3 カラムで表示（Kanban 風だがもっと静か）。

### Why
- カップル間の「今どこまで決まった」が、会話を待たず共有される
- 相手が見ている時に自分が何を思っているか分かる
- 議事録代わり、見学の帰り道で見返せる

### How
```prisma
model Agreement {
  id String @id
  projectId String
  status "decided" | "wip" | "discuss_needed"
  topic String  // "挙式スタイル", "ゲスト人数" etc
  content String?
  createdBy String (userId)
  createdAt DateTime
}
```
- 決定時 / 評価変更時に自動で「決まった」を追加（候補決定、予算決定 等）
- マイページに 3 カラム表示
- タップで /coach?topic=xxx で話す
